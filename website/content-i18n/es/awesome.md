## Contribuir

Por favor, asegúrate de que tu pull request cumpla con lo siguiente:

- Busca sugerencias anteriores antes de hacer una nueva para asegurarte de que la tuya no sea un duplicado.
- Realiza un pull request individual para cada sugerencia.
- Usa el siguiente formato: [NOMBRE](ENLACE) - DESCRIPCIÓN.
- Mantén las descripciones simples.
- Se aceptan nuevas categorías o mejoras a las existentes.
- Termina todas las descripciones con un punto.
- Intenta que tus pull requests tengan solo un commit si es posible. Puedes combinar tus commits en uno antes de enviar.
- Haz que los títulos y descripciones de tus pull requests sean lo más detallados posible. Explica por qué un recurso merece ser añadido a la lista.
- Asegúrate de que tus adiciones estén en orden alfabético. Esto incluye los encabezados y los elementos en sí.
- Asegúrate de que los recursos que envíes sean de alta calidad, estén bien documentados y se mantengan recientemente. También ayuda que sean populares para que pueda investigarlos antes de añadirlos a la lista.
- Esta lista no está pensada para anunciar tus propios recursos. Los recursos gratuitos o de código abierto deben tener prioridad sobre los de pago si tienen calidad y funcionalidades similares.
