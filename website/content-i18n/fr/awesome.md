## Contribution

Veuillez vous assurer que votre pull request respecte les points suivants :

- Recherchez les suggestions précédentes avant d'en faire une nouvelle pour éviter les doublons.
- Faites une pull request individuelle pour chaque suggestion.
- Utilisez le format suivant : [NOM](LIEN) - DESCRIPTION.
- Gardez les descriptions simples.
- Les nouvelles catégories ou les améliorations des catégories existantes sont les bienvenues.
- Terminez toutes les descriptions par un point.
- Essayez de faire en sorte que vos pull requests ne contiennent qu'un seul commit si possible. Vous pouvez regrouper vos commits en un seul avant de soumettre.
- Rendez les titres et descriptions de vos pull requests aussi détaillés que possible. Expliquez pourquoi une ressource mérite d'être ajoutée à la liste.
- Assurez-vous que vos ajouts sont classés par ordre alphabétique, y compris les en-têtes et les éléments eux-mêmes.
- Vérifiez que les ressources que vous soumettez sont de haute qualité, bien documentées et récemment maintenues. Il est également utile qu'elles soient populaires afin que je puisse les étudier avant de les ajouter à la liste.
- Cette liste n'est pas destinée à faire la publicité de vos propres ressources. Les ressources gratuites ou open-source doivent être privilégiées par rapport aux ressources payantes si elles sont de qualité et de fonctionnalités similaires.
