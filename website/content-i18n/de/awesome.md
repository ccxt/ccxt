## Mitwirken

Bitte stelle sicher, dass dein Pull Request folgendes erfüllt:

- Durchsuche vorherige Vorschläge, bevor du einen neuen einreichst, um Duplikate zu vermeiden.
- Erstelle für jeden Vorschlag einen eigenen Pull Request.
- Verwende folgendes Format: [NAME](LINK) - BESCHREIBUNG.
- Halte Beschreibungen einfach.
- Neue Kategorien oder Verbesserungen bestehender sind willkommen.
- Beende alle Beschreibungen mit einem Punkt.
- Versuche, wenn möglich, nur einen Commit pro Pull Request zu haben. Du kannst deine Commits vor dem Einreichen zusammenfassen.
- Gestalte deine Pull-Request-Titel und -Beschreibungen so detailliert wie möglich. Erkläre, warum ein Asset oder eine Ressource es verdient, der Liste hinzugefügt zu werden.
- Stelle sicher, dass deine Ergänzungen in alphabetischer Reihenfolge sind. Das gilt sowohl für die Überschriften als auch für die Einträge selbst.
- Stelle sicher, dass die von dir eingereichten Assets von hoher Qualität sind, gut dokumentiert und kürzlich gepflegt wurden. Es hilft auch, wenn sie beliebt sind, damit ich sie recherchieren kann, bevor ich sie zur Liste hinzufüge.
- Dies ist nicht dazu gedacht, als Liste zur Bewerbung eigener Assets verwendet zu werden. Kostenlose oder Open-Source-Assets sollten gegenüber kostenpflichtigen bevorzugt werden, wenn sie von ähnlicher Qualität und Funktionalität sind.
